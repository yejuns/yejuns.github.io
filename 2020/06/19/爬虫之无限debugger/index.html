爬虫之无限debugger

2020-06-19 爬虫 评论

无限debugger

无限debugger 也叫做无限断点。 在 JavaScript 中，debugger 指令就是用来添加断点的。所以，开发者在反调试的时候，会在 JavaScript 中开启一个单独的线程，来循环调用 debugger。这样一来，如果有人想要进入到调试模式，就会不断地停滞在无意义的断点处，而无法正常调试。在正常运行JavaScript 的时候，debugger 不会生效，也就不会影响用户的正常使用。

进入debugger模式

  1. 在控制台输入debugger，即可进入 debugger 模式；
  2. 通过在控制台输入 eval("debugger;alert(1234)"),我们可以进入 debugger 模式；
  3. 通过 function 实现 debugger , 在终端输入，Function('debugger').apply() 或者 Function('debugger').call()

我这边以百度为例，以上面的任意一种方式，都可以进入网页的 debugger 模式

无限debugger

实际上不可能存在真正的无限debugger，那样会造成浏览器卡死。假如某网站的无限 debugger 这样实现

在关闭 devTools 的一瞬间，整个 chrome 会被卡住，因为主线程会阻塞在 debugger 的地方，导致别的地方无法继续运行。

所以大部分是通过一个定时器去频繁地执行debugger，或者是在业务代码之前或者之后设置了几百个debugger。

对于这样的定时任务，我们可以直接注入，将相关定时任务删除。
